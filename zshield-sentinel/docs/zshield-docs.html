Training Guide (Step-by-Step)

1. Install and Activate

  1. Go to WP Admin > Plugins > Add New.
  2. Upload the ZShield Sentinel plugin ZIP.
  3. Click Install Now and then Activate.

2. Open the Dashboard

  1. Go to WP Admin > ZShield > Dashboard.
  2. Review the Security Score and Protection Status.
  3. Use quick actions to jump to key pages.

3. Configure Security Settings

  1. Open Settings.
  2. Enable Login Guard and set attempts + lockout time.
  3. Enable 2FA + Login Captcha if you want stronger authentication.
  4. Configure Login Hardening with access key or custom login slug.
  5. Enable Firewall to block common exploit patterns.
  6. Enable Security Headers, XML-RPC Disable, and Pingbacks Disable.
  7. Enable Block Author Enumeration.
  8. Disable the File Editor and hide WP version.
  9. Click Save Changes.

4. Create File Integrity Baseline

  1. Make sure the site is clean and updated.
  2. Go to File Integrity.
  3. Click Create Baseline.

Only recreate baseline after trusted updates.

Optional: Enable Core File Integrity if you also want to monitor WordPress core files.

5. Run Manual Scans

  1. Go to File Integrity.
  2. Click Run Scan.
  3. Open Diff Preview for modified files.

6. Malware Scan

  1. Go to Malware Scan.
  2. Click Run Malware Scan.
  3. Filter by severity and review findings.
  4. Use Quarantine to isolate suspicious files.

Always confirm false positives before action. Enable scheduled malware scans if you want automatic checks.

7. Audit Log Review

  1. Go to Audit Log.
  2. Check recent events (failed logins, lockouts, scans).
  3. Use Export CSV if you need offline reports.
  4. Investigate unusual activity quickly.

8. Configure Email Alerts

  1. Enable Email Alerts for lockouts.
  2. Set the Alert Email.
  3. Enable Scan Email Reports if required.

Step-by-Step Workflow

  1. Install and activate the plugin.
  2. Review security settings and save changes.
  3. Create a baseline when everything is clean.
  4. Run manual scans after updates or changes.
  5. Check the audit log weekly for unusual activity.

When to Recreate Baseline

Recreate baseline only after you intentionally update themes or plugins and have verified the site is clean. This keeps diffs accurate.

Tip: Keep a backup before baseline creation.

Login Protection (2FA + Captcha)

  1. Enable Two-Factor Authentication in Settings.
  2. Each user can enable 2FA in their profile and add the secret to an authenticator app.
  3. Enable Login Captcha (math or reCAPTCHA) to block bots.

Login URL Hardening

  1. Enable Login URL Hardening in Settings.
  2. Set a Custom Login Slug or a Login Access Key.
  3. Use the new login URL for all users.

Tip: Save the new URL in a password manager.

Firewall Rules

  1. Enable the Application Firewall.
  2. Add allowlist or blocklist IPs if needed.
  3. Exclude safe paths to reduce false positives.

Role-Based Access

  1. Set Minimum Access Role in Settings.
  2. Only users with that role (or higher) can access ZShield pages.

Settings Import/Export

  1. Go to Settings.
  2. Use Export Settings (JSON) to save a backup.
  3. Use Import Settings to apply a saved configuration.

Malware Quarantine

  1. Review a finding in Malware Scan.
  2. Click Quarantine to isolate the file.
  3. Restore it later if confirmed safe.

How to Install the Plugin

  1. Download the ZShield Sentinel plugin ZIP.
  2. Go to WP Admin > Plugins > Add New.
  3. Click Upload Plugin and choose the ZIP file.
  4. Click Install Now.
  5. Click Activate after installation completes.

Alternative (FTP Upload)

  1. Extract the ZIP file.
  2. Upload the folder to wp-content/plugins/.
  3. Go to Plugins and click Activate.

Understanding Reports

Report Type What It Means Action
Added File A new file appeared that was not in baseline. Verify if it came from an update or a plugin install.
Modified File A file changed since baseline. Use diff preview to inspect changes.
Removed File A file was deleted after baseline. Confirm if removal was expected.

Malware Scan Controls

Fine-tune what the scanner checks to reduce noise without losing protection.

  • Enable Ignore Vendor JS to skip minified bundles.
  • Add Rule Allowlist to ignore known safe detections.
  • Add Path Allowlist for trusted directories.
  • Include Uploads and MU-Plugins for wider coverage.
  • Use Quarantine to safely isolate suspicious files.

Uninstall Cleanup

Turn on Remove Data on Uninstall if you want all logs and settings deleted when the plugin is removed.

Tip: Keep it off if you want to preserve audit history.

Troubleshooting

Too Many Scan Changes

If scans report many changes, confirm recent updates. Recreate baseline only after verifying changes are safe.

Emails Not Received

Check your SMTP or email provider settings. Test by sending a WordPress email.

Lockouts Triggered

Whitelist your IP if needed and review failed login attempts in the audit log.

Diff Preview Not Available

Diffs are available only for files stored in baseline. Recreate baseline to enable new diffs.

Cannot Access Login Page

If Login URL Hardening is enabled, use the custom slug or access key. Disable the setting if you lost access.

Captcha Failures

For reCAPTCHA, verify your site/secret keys. For math captcha, refresh the login page to get a new challenge.

FAQ

Does this plugin replace a firewall?

It includes an application firewall for common patterns, but a dedicated WAF can provide extra protection.

How often should I run a scan?

Weekly is enough for stable sites. Run a manual scan after updates.

Can I export audit logs?

Yes. Use the Export CSV button on the Audit Log page.

How does 2FA work?

Enable 2FA globally, then each user can activate it in their profile using any TOTP authenticator app.

What This Plugin Does

  • Protects login with throttling and lockouts.
  • Adds 2FA (TOTP) and captcha to harden login.
  • Provides login URL hardening with custom slug or access key.
  • Blocks author enumeration attempts to reduce user discovery.
  • Adds security headers and disables risky features.
  • Disables XML-RPC and pingbacks to shrink attack surface.
  • Hides WordPress version and disables the file editor.
  • Applies firewall rules to block common exploit patterns.
  • Tracks sensitive changes in the audit log.
  • Creates a baseline of trusted files and checks for tampering.
  • Optionally includes WordPress core files in integrity scans.
  • Runs malware scans to detect suspicious code patterns.
  • Quarantines suspicious files with restore support.
  • Exports/imports settings for quick setup.
  • Enforces role-based access to plugin controls.
  • Provides a security score to summarize posture.
